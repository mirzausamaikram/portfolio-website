Musical Ensemble Management System (MEMS) – OOP & Design Patterns

A comprehensive console-based application demonstrating advanced design pattern implementation and object-oriented principles in Java. MEMS enables users to create and manage two ensemble types (Orchestra and Jazz Band), each with role-specific musicians, while supporting full undo/redo functionality across all operations. The system leverages the **Abstract Factory pattern** to encapsulate ensemble and role creation, allowing seamless extension for new ensemble types. The **Command pattern** forms the architectural backbone—every operation (create, add, modify, delete, rename) is encapsulated as a Command object, enabling clean separation of concerns and straightforward operation history management. The **Memento pattern** is integrated within command objects to capture and restore ensemble state, preserving names and musician roles during undo/redo operations. The **Invoker class** maintains separate stacks for undo and redo commands, orchestrating execution and rollback seamlessly. This pattern composition demonstrates how well-designed architecture enables powerful features without code duplication. Built with pure Java using OOP principles (encapsulation, inheritance, polymorphism) and leveraging Vector collections for type-safe storage.
